# 自定义周期功能说明

## 功能概述

新增了"自定义周期"类型，允许用户使用计划开始时间和计划结束时间来定义目标周期。

## 更改内容

### 1. 数据库层
- **Prisma Schema**: 在 `PeriodType` 枚举中添加了 `CUSTOM` 类型
- **数据库**: 已同步到 PostgreSQL 数据库

### 2. 类型定义
- **types/index.ts**: 更新 `PeriodType` 类型包含 `'CUSTOM'`

### 3. 验证层
- **server/validations/goal.ts**: 更新 `periodBaseSchema` 验证规则
- **lib/validations/period-utils.ts**:
  - 添加 CUSTOM 类型到 `PeriodType`
  - 扩展 `getPeriodLabel` 函数支持自定义周期标签显示
  - 更新 `getCurrentPeriodValue` 函数
  - 更新 `validatePeriodValue` 函数

### 4. UI 组件
- **components/PeriodSelector.vue**:
  - 添加"自定义周期"选项到下拉菜单
  - 当选择自定义周期时，显示提示信息引导用户设置计划开始/结束时间
  - 添加样式美化提示信息
- **components/GoalModal.vue**: 更新类型定义

## 使用方法

### 在目标创建/编辑界面：

1. **选择周期类型**: 在"周期"下拉菜单中选择"自定义周期"
2. **设置时间范围**:
   - 在"时间规划"部分设置"计划开始时间"
   - 在"时间规划"部分设置"计划结束时间"
3. **保存**: 保存目标后，系统会自动使用开始和结束时间作为自定义周期

### 显示格式：

自定义周期会以以下格式显示：
```
YYYY-MM-DD 至 YYYY-MM-DD
```

例如：`2024-01-01 至 2024-12-31`

## 与其他周期的区别

| 周期类型 | 需要设置 | 显示示例 |
|---------|---------|---------|
| 年目标 | 年份 | 2024年 |
| 月目标 | 年-月 | 2024年01月 |
| 周目标 | 年-周 | 2024年第1周 |
| 任务 | 日期 | 2024年01月01日 |
| **自定义周期** | **计划开始/结束时间** | **2024-01-01 至 2024-12-31** |

## 开发服务器

当前运行在: http://localhost:3007

## 注意事项

- 自定义周期必须设置计划开始时间和计划结束时间
- 结束时间必须晚于开始时间
- periodValue 对于 CUSTOM 类型固定为 "CUSTOM"
