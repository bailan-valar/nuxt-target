# 笔记拖拽排序和置顶功能

## 已实现的功能

### 1. 拖拽排序
- 使用 `sortablejs` 库实现笔记卡片的拖拽排序
- 拖拽完成后自动调用 `/api/notes/reorder` API 更新所有笔记的 `sortOrder` 值
- 排序时使用平滑的动画效果（150ms）

### 2. 置顶功能
- 点击置顶按钮会调用新的 `/api/notes/{id}/pin` API
- 置顶操作会将笔记的 `sortOrder` 设置为当前最小值 - 1，确保置顶的笔记排在最前面
- 不再使用单独的置顶分组，而是通过 `sortOrder` 实现排序

## 技术实现

### 前端组件 ([components/NoteList.vue](components/NoteList.vue))
- 添加 `sortOrder` 字段到 `Note` 接口
- 修改 `sortedNotes` 计算属性，使用 `sortOrder` 作为主要排序字段
- 集成 `sortablejs` 库实现拖拽功能
- 在 `onMounted` 中初始化拖拽排序
- 拖拽结束时调用 `/api/notes/reorder` 批量更新排序

### 后端 API
- `/api/notes/{id}/pin` (PUT) - 新增置顶/取消置顶接口
  - 获取当前用户笔记列表中最小的 `sortOrder`
  - 将笔记的 `sortOrder` 设置为最小值 - 1
  - 同时切换 `isPinned` 状态
- `/api/notes/{id}.put.ts` - 更新支持 `sortOrder` 字段
- `/api/notes/index.get.ts` - 修改排序逻辑，按 `sortOrder` 升序排序
- `/api/notes/reorder.put.ts` - 已存在，批量更新笔记排序

## 数据库
- `Note` 模型已有 `sortOrder` 字段（integer，默认为 0）
- 创建新笔记时自动分配下一个可用的 `sortOrder` 值

## 使用说明

### 拖拽排序
1. 在笔记列表中，鼠标按住任意笔记卡片
2. 拖动到目标位置
3. 松开鼠标，笔记会自动保存到新位置

### 置顶笔记
1. 点击笔记卡片右上角的图钉图标
2. 笔记会立即移动到列表最上方
3. 再次点击可取消置顶
